🎯 What It Does

Students with executive function difficulties struggle to turn a pile of syllabi and calendar invites into an actionable study plan. CEF automates that translation.

📄

Ingest syllabi & documents

Upload PDFs, DOCX files, or Google Drive links. The AI reads every deadline, exam date, and assignment weight automatically.

🗓️

Build the master calendar

All extracted events are de-duplicated and merged with your existing Google Calendar into one unified schedule.

📚

Generate study plans

The AI creates proactive study blocks proportional to each deliverable's grade weight, respecting your configured work hours and break rules.

💬

Chat with your courses

Ask questions against all loaded syllabi — the context agent retrieves the most relevant fragments and keeps conversation history.

✂️

Break down tasks

Any deadline can be recursively decomposed into subtasks up to 3 levels deep via the "Break It Down" dialog.

🔄

Two-way sync

Changes in Google Calendar are pulled back into the app; local edits are pushed out. Conflicts surface an interactive proposal diff before anything is written.

🤖 Agent Layer

Four specialized agents coordinate ingestion, event generation, dedup, and calendar sync.

IngestionAgent

Reads raw source files and URLs and extracts high-fidelity SourceFragments. Calls categorizeSource() on every non-ICS source to tag it by type (syllabus, lecture, handout…).

EventAgent

Consumes structured fragments and calls Gemini to produce both Deliverable events (exams, assignments) and proactive Study Plan blocks, weighted by grade percentage.

NormalizationService

Programmatic safety net: deduplicates events via SHA-256 content hashing, standardizes category labels, and enforces 9–21 hr study-hour constraints before persistence.

CalendarAgent

Intelligent sync gateway. Manages OAuth refresh, pagination, and conflict detection. Surfaces a SyncNegotiationDialog so the student approves proposal diffs before any write.

ContextAgent

Multi-source retrieval for the chat panel. Scores fragments with TF-IDF + recency, injects the top-k into the Gemini prompt, and maintains conversation history.

AgentHarness

Active lifecycle driver. Runs the full agent pipeline at startup and once per day, ensuring the calendar stays fresh without any manual "Sync" tap from the student.

📱 UI Screens

Academic Calendar

Central dashboard. Events grouped by date, colour-coded by category. Shows countdown chips, study progress bars, and a Semester Health card summarising upcoming workload.

Sources Panel

Manage all inputs — local files, web URLs, and Google Drive. Each source shows its sync/analysis status and can be re-processed on demand.

Studio Panel

AI workspace. Chat with your loaded syllabi, trigger manual analysis, and review AI-generated study blocks before they land on the calendar.

Settings

Connect your Google account, enter your Gemini API key, configure preferred study hours and break rules, and select which Google Calendar to sync into.

Task Decomposition Dialog

"Break It Down" — recursively decomposes any DEADLINE or FINALS event into subtasks up to depth 3 via the Critic-Actor AI loop.

Sync Negotiation Dialog

Interactive proposal diff that replaces silent conflict resolution. The student approves, rejects, or merges each proposed calendar change before it is written.

✨ AI Strategy

Primary Engine

Gemini 1.5 Flash via direct REST API — 1 M token context window, chosen for stability and the ability to ingest an entire syllabus in one call.

Privacy-First

Students supply their own free Gemini API key from Google AI Studio. Documents never leave the student's Google ecosystem.

Model Auto-Negotiation

ModelManager caches the best available model in SQLite and retries automatically against a fallback preference list on quota exhaustion.

Confabulation Gate

GroundingGuardAIService filters every AI-generated event against years explicitly mentioned in the source text — silently dropping dates that can't be grounded.

Critic-Actor Loop

Complex tasks (study plans, decompositions) run through CriticActorAIService with graph-based cycle detection to prevent oscillation.

Source Fact Grounding

SourceFactGrounder extracts date and grade-weight claims from free-text AI output and cross-checks each against the source text, appending structured warnings for ungrounded claims.

Key Patterns

StateFlowReader / StateFlowWriter

Raw StateFlow<T> is never exposed publicly. Read-only and write-only interfaces separate concerns, making every reactive dependency trivially mockable in tests.

Thin Facade + Service Extraction

Any class coordinating 3+ responsibilities delegates to focused single-responsibility services injected via constructor. The facade itself stays under complexity 5.

Deterministic Event IDs

Events are keyed by SHA-256 hash of their content fields rather than random UUIDs — preventing duplicate entries on every save.

Integration Test Naming Gate

Any test that makes real API calls must carry the IntegrationTest suffix. Default Gradle run excludes these; -PrunAITests=true enables them.

⚙️ Setup & Build

Prerequisites

JDK 17+, Android SDK, Xcode (for iOS), and a free Gemini API key from Google AI Studio. Google Cloud project with Calendar + Drive APIs enabled.

Environment

Copy .env_template → .env and populate GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET. The Gemini API key is entered per-user at runtime in Settings, not in the build.

Run Profiles

local — real Google Calendar & OAuth. test — mock calendar, deterministic data, no network required.

./gradlew :composeApp:assembleDebug          # Android
./gradlew :composeApp:run                    # Desktop (JVM)
# Open iosApp/ in Xcode for iOS
./gradlew :composeApp:jvmTest                # Unit tests (no API calls)
./gradlew :composeApp:jvmTest -PrunAITests=true  # Include integration tests
